# ZwiCoat Website — Project Overview for Claude Code

## What is this project?

The public-facing website for ZwiCoat Materials Innovations, LLC (ZCMI). Built with React + Vite. Deployed as a static site on an AWS EC2 server shared with the Dumbo Press website.

---

## Tech Stack

- **Framework:** React 18 + Vite 5
- **Routing:** React Router DOM v6
- **Styling:** Styled-components (wrappers in `src/assets/wrappers/`) + global CSS in `src/index.css`
- **UI libs:** Swiper, Slick carousel, React Icons, styled-components

---

## Folder Structure

```
ZwiCoat website/
├── src/
│   ├── pages/          # One file per page (Landing, About, Technology, Contact, FCOI, HomeLayout)
│   ├── components/     # Reusable components (Navbar, Footer, Hero, TeamList, etc.)
│   ├── assets/
│   │   ├── wrappers/   # Styled-component wrappers for each component
│   │   └── images/     # All image assets
│   ├── data.js         # Team member data and other static content
│   ├── App.jsx         # Router config — add new routes here
│   ├── main.jsx        # Entry point
│   └── index.css       # Global CSS variables and base styles
├── public/             # favicon, logo
├── dist/               # Built output (generated by `npm run build`) — NOT committed to git
└── CLAUDE.md           # This file
```

---

## Adding a New Page

1. Create `src/pages/MyPage.jsx`
2. Export it from `src/pages/index.js`
3. Add a route in `src/App.jsx` inside the `children` array of the root route
4. Optionally add a `NavLink` in `src/components/Navbar.jsx`

## CSS / Styling Conventions

- Color variables are defined in `src/index.css` under `:root` — use them (`--primary-1`, `--grey-darkest`, etc.) rather than hardcoded colors
- Each component has a corresponding styled-component wrapper in `src/assets/wrappers/`
- The FCOI page (`src/pages/FCOI.jsx`) uses inline styles as an exception — it's a plain document page

### Apache config location
`/etc/apache2/sites-available/zwicoat.conf`

Apache serves the `dist/` folder as static files. `FallbackResource /index.html` handles React Router client-side routing.

### Restarting Apache (rarely needed)
```bash
sudo systemctl reload apache2
```

### Checking Apache errors
```bash
sudo tail -30 /var/log/apache2/error.log
sudo apache2ctl configtest
```

## Things to Be Careful About

- **Always `npm run build` before deploying** — catch errors locally, not on the live site
- **`dist/` is gitignored** — the build runs on EC2, not committed to git
- **Apache serves static files** — there is no running server process to restart after a deploy; just rebuild
- **Shared EC2 with Dumbo Press** — both sites run on the same server. Avoid changes to global Apache config (`apache2.conf`) without understanding the impact on Dumbo Press
- **Direct IP access is blocked** — `http://3.131.73.139` returns Forbidden intentionally (disable-ip.conf)
